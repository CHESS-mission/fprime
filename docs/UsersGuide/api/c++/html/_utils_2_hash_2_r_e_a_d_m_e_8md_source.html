2 # Utils::Hash
3 
4 This directory contains a generic interface for creating hashes
5 of data. It provides a mechanism for selecting different implementations
6 of the hash interface, which could be as simple as an 8 bit checksum or as
7 complex as a 256 bit SHA256 hash.
8 
9 The generic interface to the hashing utility is included in `Utils/Hash/Hash.hpp`.
10 Its corresponding generic hash buffer class (which is used to hold a computed hash
11 digest) can be found in `Utils/Hash/HashBuffer.hpp`. Some common implementation code
12 for each class is included in `Utils/Hash/HashCommon.cpp` and `Utils/Hash/HashBufferCommon.cpp`.
13 
14 Specific implementations of the hashing utility are stored in subdirectories in `Utils/Hash/`.
15 Currently, one such implementation exists in `Utils/Hash/openssl/` which provides a SHA256
16 hash using the openssl library. Another implementation is also provided which calculates a
17 32-bit CRC32, which depends on no external libraries.
18 
19 A specific implementation can be selecting by modifying the `HashConfig.hpp` file.
20 
21 ## Using `hash`
22 
23 The generic hash interface includes only 4 methods besides the constructor/destructor. A
24 description of each method is included below:
25 
26 `hash.init()` - This method initializes the hash object, priming it for computing a new hash. It
27 should be run before beginning to calculate each new hash. It resets any state remaining
28 from a previously computed hash.
29 
30 `hash.update(data, len)` - This method updates the hash object with new data to hash. It can be run as many
31 times as you like, allowing you to continue adding data to the hash state. In this way, a user can hash their data as they read it
32 from a buffer, or from a file, in segments.
33 
34 `hash.final(buffer)` - This method returns a hash of all the data given to the hash object via `update` since
35 the last `init` was run. It returns the hash in `buffer`, which is a `HashBuffer` object.
36 
37 `hash.hash(data, len, buffer)` - This method runs `init`, `update`, and `final` all in one shot. If you have already
38 collected all the data that you want to hash into a buffer `data` with length `len`, you can use this static function
39 to calculate the hash all at once. The computed hash is returned in `buffer`, which is a `HashBuffer` object.
40 
41 ## Configuring `hash`
42 
43 To configure the `hash` utility to use a specific hashing implementation, modify `HashConfig.hpp` to include
44 the header file of the implementation you wish to use. For example, to use SHA256 hashing, your `HashConfig.hpp`
45 should look something like:
46 
47 ```
48 #ifndef UTILS_HASH_CONFIG_HPP
49 #define UTILS_HASH_CONFIG_HPP
50 
51 #include <Utils/Hash/openssl/SHA256.hpp>
52 
53 #endif
54 ```

56 ## Building your own `hash` implementation
57 
58 The generic interface to `hash` can be implemented using many different hashing algorithms. To construct your own
59 please look at the SHA256 implementation as an example, found in the `openssl` subdirectory. Your implementation
60 should include a few files:
61 
62 `YourImplementationHash.hpp` - This file must declare the following `#define` constants: `HASH_HANDLE_TYPE`, the handle type for computing the hash internally, `HASH_DIGEST_LENGTH`, the length of the hash digest in bytes, and `HASH_EXTENSION_STRING`, the file extension you would assign to the hash should it be stored into a file (ie. `.SHA256`).
63 
64 `YourImplementationHash.cpp` - The implementation file for the hashing algorithm. It should implement the methods declared
65 in `Hash.hpp`, except for those included in `HashCommon.cpp`.
66 
67 `README.md` - So that those who come after you will understand how your implementation works and what it is used for ;)
