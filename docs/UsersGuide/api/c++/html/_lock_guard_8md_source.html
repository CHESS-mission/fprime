2 # Utils::LockGuard
3 
4 ## 1 Introduction
5 
6 `LockGuard` is a RAII-wrapper around Os::Mutex, essentially allowing scoped
7 lock and unlock. It works the same way as C++11 `std::lock_guard`.
8 
9 If a section of code has only basic lock/unlock requirements, `LockGuard`
10 ensures that early exits from that section will always release the lock.
11 
12 ## 2 Usage
13 
14 ### 2.1 Basic
15 
16 LockGuard is a class that can be included and instantiated with an associated
17 Os::Mutex reference.
18 
19 ```cpp
20 #include <Utils/LockGuard.hpp>
21 ...
22 Os::Mutex mutex;
23 LockGuard lock(mutex);
24 ```
25 
26 The moment LockGuard is instantiated, it will attempt to lock the mutex that's
27 given to it.
28 
29 LockGuard is meant for scoped use, i.e. it should always live on a function
30 call stack. It allows a sometimes more convenient way of using mutex, e.g. you
31 want this entire function locked from this point forward, and early returns
32 should be guaranteed to release the lock.
33 
34 ```cpp
35 LockGuard _lock(mutex);
36 ...
37 if (xyz) {
38  return 0; // will unlock mutex
39 }
40 if (abc) {
41  return 1; // will unlock mutex
42 }
43 return 2; // will unlock mutex
44 ```
45 
46 It can also be used for a smaller scope
47 
48 ```cpp
49 if (xyz) {
50  LockGuard _lock(mutex);
51  ...
52 } // will unlock mutex
