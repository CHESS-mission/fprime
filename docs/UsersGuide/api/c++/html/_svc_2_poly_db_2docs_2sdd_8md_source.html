2 # Svc::PolyDb Component
3 
4 ## 1. Introduction
5 
6 The `Svc::PolyDb` component is meant to act as a resident database of values that components can use for checking or for data in algorithms.
7 
8 ## 2. Requirements
9 
10 The requirements for `Svc::PolyDb` are as follows:
11 
12 Requirement | Description | Verification Method
13 ----------- | ----------- | -------------------
14 PDB-001 | The `Svc::PolyDb` component shall store `Fw::PolyType` values. | Inspection, Unit Test
15 PDB-002 | The `Svc::PolyDb` component shall allow the `Fw::PolyType` values to be read and written. | Unit Test
16 PDB-003 | The `Svc::PolyDb` component shall time tag the data | Unit Test
17 PDB-004 | The `Svc::PolyDb` component shall report the measurement state of the data (good, stale, failure) | Unit Test

38 #### 3.2 Functional Description
39 
40 `Fw::PolyType` is different from binary telemetry in that it is not in a serialized form, but is stored as the native type.
41 The component stores a table of `Fw::PolyType' objects which are read and written by table index.
42 The table is protected by a mutex to prevent simultaneous access.
