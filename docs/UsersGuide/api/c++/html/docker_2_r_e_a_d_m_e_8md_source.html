1 # F´Docker
2 
3 Docker is a containerization service. It allows users to pack dependencies into a "container" and
4 run it as a packaged application. It functions like a lightweight VM, but at its core it wraps
5 OS calls allowing for more efficient sharing of resources with the host server.
6 
7 F´ uses docker to wrap the dependencies for building and running F´. From this, F´can be deployed
8 to servers and computers easily in a packaged standard format. This is specifically useful for
9 things like Continuous Integration where some unknown worker will need to build and run F´ without
10 the need to ensure every worker has F´ build dependencies installed.
11 
12 ## Docker Build and Publish Instructions
13 
14 Our Dockerfile is organized into substages. The first stage is setup with just the tools installed
15 and the entrypoint is setup to be a generic `bash` prompt. This stage may be used for projects
16 that want their developers using Docker as an environment. The other stage is the Jenkins-CI stage
17 that adds in the needed user setup for running continuous integration on Jenkins CI. **Note:**
18 F´ CI using GitHub actions is done off from the base image, as these Actions do not require extensive
19 user setup like Jenkins.
20 
21 ### Building Docker Images
22 
23 This section will describe building each Docker image in turn. In the end, the user will have built
24 the `latest` docker images and these will be available for users to run.
25 
26 **Note:** this base does not require a context (files from the local machine) to build but docker
27 will "upload" one anyway. Thus we `cd` into the directory in order to assure that we are uploading
28 as few files as possible.
29 
30 **Build fprime-base**
31 ```
32 cd fprime/mk/docker
33 docker build --target fprime-base -t fprime-base:latest .
34 ```
35 
36 **Build fprime-ci**
37 ```
38 cd fprime/mk/docker
39 docker build --target jenkins-ci -t fprime-base:latest .
40 ```

55 ## Dockerfile
56 
57 Docker uses a Dockerfile to describe the container that will be built. It is like a recipe for
58 setting up a blank machine (Ubuntu, etc) into a dependency compliant machine. The F´ supplied
59 Dockerfile uses Ubuntu 18.04 as a base and installs the required dependencies needed to build F´.
60 The result is a containerized F´ builder for use with CI.
61 
62 ### Dockerfile Explained
63 
64 Dependencies are installed in one step. This includes an update and install of Ubuntu
65 packages, running the ubuntu-packages script, and install the pip dependencies. Finally, the the
66 app-cache is cleaned. Doing all this in one step prevents excessive Docker image size bloat by
67 reducing the number of temporary file system layers.
68 
69 The user "jenkins" is created, given root permissions, and set as the user for when the
70 container is run. The PATH and ENTRYPOINT is set for when the container runs. Currently, the ENTRYPOINT
71 is set to a `bash` shell such that the container will provide a terminal and "run forever", which is
72 needed for Jenkins CI.
73 
74 ## Manual Docker Usage
75 
76 Should a user need to run this docker image for purposes other than CI, docker can be run manually to
77 create an F´container. The user will be root when running the `fprime-base` image. Please ensure that
78 the fprime-base image has been built or downloaded as latest. If downloaded, run with the tag
79 `nasafprime/fprime-base:latest` instead of just `fprime-base:latest`. An fprime checkout should be
80 mapped into the container.
81 
82 **Note:** the user/developer should understand Docker UUIDs and the implications in mounting volumes
83 into a running Docker container because this differs accross host OSes.
84 
85 ### Run the F´Docker Image Manually
86 ```bash
87 fprime>docker run -it -v <path to fprime checkout>:/fprime fprime-base:latest
88 docker> cd /fprime
89 ... Install an F´ venv ...
90 ... F´ commands here ...
91 ```
92 Here the container is run interactively with a pseudo-terminal and F´checkout is mounted to
93 `/fprime`. The user can then do an F´ install and then run F´ commands with the RPI tools and OS
94 dependencies installed.
